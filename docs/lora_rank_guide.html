Rank vs Quality Tradeoff

When to Use Low Rank (1-8)

  • Simple tasks: Classification, NER, sentiment analysis
  • Small datasets: < 1K samples (low rank prevents overfitting)
  • Memory constrained: Consumer GPUs (8-16GB)
  • Quick iteration: Prototyping and experimentation
  • Multi-adapter: Serving many task-specific adapters

When to Use Medium Rank (16-32)

  • Instruction tuning: Chat, QA, summarization
  • Medium datasets: 1K-100K samples
  • Good balance: Quality vs speed sweet spot
  • Most use cases: Default recommendation
  • Production: When quality matters but cost does too

When to Use High Rank (64-128)

  • Complex tasks: Reasoning, code generation, math
  • Large datasets: > 100K samples
  • Domain adaptation: Medical, legal, scientific
  • Maximum quality: When compute is not a constraint
  • Close to full FT: When LoRA must match full fine-tuning

Common Mistakes

  • Too high rank: Overfits on small datasets, wastes compute
  • Too low rank: Underfits on complex tasks
  • Wrong alpha: alpha should be 2x rank (scaling factor)
  • Wrong targets: Always target q_proj and v_proj minimum
  • Ignoring data: More data = can use higher rank safely
