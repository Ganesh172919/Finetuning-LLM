RLHF — Reinforcement Learning from Human Feedback

How it works: Train a reward model on human preferences, then use PPO to optimize the policy against the reward model.

Pipeline: SFT → Reward Model Training → PPO Optimization

Pros: Most flexible, works with any objective

Cons: Complex, 3-4x memory, reward hacking risk

Used by: ChatGPT, Claude, InstructGPT

DPO — Direct Preference Optimization

How it works: Skip the reward model entirely. Optimize policy directly on preference pairs using a logistic loss.

Loss: L = -log σ(β · (log π(y_w)/π_ref(y_w) - log π(y_l)/π_ref(y_l)))

Pros: Simple, no reward model, stable training

Cons: Needs preference pairs, can overfit

Used by: Zephyr, many open-source models

GRPO — Group Relative Policy Optimization

How it works: Generate N responses per prompt, score them automatically, optimize to prefer higher-scoring responses.

Key insight: Group mean is the natural baseline — no value function needed!

Pros: No preference data needed, works with automatic rewards

Cons: Needs good reward function, more compute per step

Used by: DeepSeek-R1, DeepSeek-Math

PRM — Process Reward Model

How it works: Score each STEP of reasoning, not just the final answer. Enables step-level selection and error detection.

Use cases: Best-of-N selection, training signal, error localization

Pros: Dense feedback, catches errors early

Cons: Needs step-level labels (expensive)

Used by: OpenAI o1, DeepSeek-R1, Math-Shepherd
