The Problem: Naive Autoregressive Generation

Without KV cache, generating each new token requires recomputing attention for ALL previous tokens. This is O(T²) compute per token — extremely wasteful.

How KV Cache Works

Without KV Cache

# Generate token T+1 for position in range(T+1): # Recompute K, V for ALL positions Q = W_q @ x[position] K = W_k @ x[position] V = W_v @ x[position] # Attention over ALL previous attn = softmax(Q @ K.T / √d) output = attn @ V # Total: O(T²) per token! # Wastes 90%+ of computation

With KV Cache

# Generate token T+1 # Only compute for NEW token Q = W_q @ x[T] K_new = W_k @ x[T] V_new = W_v @ x[T] # Append to cache cache.K.append(K_new) cache.V.append(V_new) # Attention using cached K, V attn = softmax(Q @ cache.K.T / √d) output = attn @ cache.V # Total: O(T) per token! # 10x faster for long sequences

The Memory Tradeoff

KV cache trades memory for compute. For long sequences, this becomes the primary memory bottleneck in LLM serving.

KV Cache Optimization Techniques

Multi-Query Attention (MQA)

Share K, V across all attention heads. Only Q is per-head. Reduces KV cache by n_heads factor (e.g., 32x smaller).

KV cache: 32× smaller

Grouped-Query Attention (GQA)

Groups of heads share K, V. Balance between MQA and MHA. Used in Llama 2 70B, Mistral 7B.

KV cache: 4-8× smaller

Paged Attention

Virtual memory for KV cache. Non-contiguous blocks. Eliminates fragmentation. Used in vLLM.

Utilization: 96%+ (vs 40%)

KV Cache Quantization

Quantize cached K, V to INT8 or INT4. Minimal quality loss. 2-4x memory reduction.

Memory: 2-4× smaller

Implementation

class KVCache: """ Key-Value Cache for efficient autoregressive generation. Stores computed K, V tensors so they don't need to be recomputed when generating the next token. Memory per layer: 2 × n_heads × head_size × seq_len × dtype_size For Llama-7B: 2 × 32 × 128 × 4096 × 2 bytes = 64 MB per layer Total: 32 layers × 64 MB = 2 GB """ def __init__(self, n_layers, n_heads, head_size, max_len): self.cache = {} for layer in range(n_layers): self.cache[layer] = { 'K': np.zeros((max_len, n_heads, head_size)), 'V': np.zeros((max_len, n_heads, head_size)), } self.seq_len = 0 def append(self, layer, k, v): # Add new K, V to cache self.cache[layer]['K'][self.seq_len] = k self.cache[layer]['V'][self.seq_len] = v def get(self, layer): # Return cached K, V up to current position return ( self.cache[layer]['K'][:self.seq_len], self.cache[layer]['V'][:self.seq_len], ) def step(self): # Advance sequence position self.seq_len += 1
