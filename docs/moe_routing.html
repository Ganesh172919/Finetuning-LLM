Mixture of Experts

How sparse models route tokens to specialized experts — scaling capacity without scaling compute

What is Mixture of Experts?

Hospital Analogy

When you visit a hospital, you don't see every doctor — a receptionist (the router) sends you to the right specialist. MoE works the same way: a gating network routes each token to the most relevant expert network.

Sparse = Efficient

A model with 8 experts might have 8× the parameters, but each token only uses 1-2 experts. So the compute cost stays similar to a small model while the knowledge capacity matches a much larger one.

Specialization

Different experts learn to handle different types of input — one might specialize in code, another in math, another in poetry. This emerges naturally during training without explicit programming.

How MoE Routing Works

The Gating Mechanism

For each input token x, the router computes a score for each expert:

G(x) = softmax(W_g · x)

Where W_g ∈ ℝE×d is the learned gating matrix, E = number of experts, d = hidden dimension.

Only the top-k experts are activated (typically k=1 or k=2):

y = Σᵢ G(x)ᵢ · Eᵢ(x) (sum over top-k experts only)

This means 6/8 or 7/8 of expert parameters are idle for each token — that's the "sparse" in sparse MoE.

Load Balancing & Capacity

Load Imbalance Problem

Without intervention, the router may collapse to always picking the same 1-2 experts ("expert collapse"). Solutions include auxiliary load-balancing losses and capacity factors.

L_aux = α · Σᵢ fᵢ · Pᵢ

Where fᵢ = fraction of tokens routed to expert i, Pᵢ = mean routing probability for expert i.

Capacity Factor

Each expert has a maximum number of tokens it can process per batch. If an expert is overloaded, excess tokens are dropped or routed to the next-best expert.

capacity = (tokens / E) · capacity_factor

Typical capacity_factor: 1.0–1.5. Higher = less dropping but more memory.

Expert Parallelism

In distributed training, each expert can live on a different GPU. Tokens are dispatched to the appropriate GPU, processed, and returned. This requires All-to-All communication.

Research Frontiers

Sparse MoE Architectures

Switch Transformer: Top-1 routing, simpler but effective.
GShard: Top-2 with capacity factors.
Mixtral: Top-2 with 8 experts per layer.
DeepSeek-V2: Shared + routed experts with fine-grained segmentation.

Soft MoE

Instead of hard top-k routing, Soft MoE computes a weighted combination of all expert outputs. Differentiable but loses the compute savings of sparse activation.

Expert Choice Routing

Instead of tokens choosing experts, experts choose tokens. Each expert picks its top-k tokens. Guarantees perfect load balance but may drop some tokens entirely.

Multi-Latent Attention (MLA)

DeepSeek-V2's innovation: compresses KV cache using low-rank decomposition, combining MoE efficiency with attention memory savings. State-of-the-art for inference.
