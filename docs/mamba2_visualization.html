📚 What is Mamba?

Mamba is a selective state space model that processes sequences in O(n) time instead of O(n²) like attention. It maintains a compressed hidden state that selectively remembers or forgets information based on the input.

h_t = A̅ · h_{t-1} + B̅ · x_t     (state update)     →     y_t = C_t · h_t     (output)

The "selective" part: B, C, and dt are input-dependent — the model learns WHAT to remember and WHEN to remember it.

Transformer (Attention)

  • Complexity: O(n²) — every token attends to every other token
  • Memory: O(n²) — stores all key-value pairs
  • Quality: Excellent — full context at every layer
  • Inference: O(n) per token with KV cache
  • Best for: Tasks needing precise token recall

Mamba (Selective SSM)

  • Complexity: O(n) — linear scan through sequence
  • Memory: O(1) per step — fixed-size hidden state
  • Quality: Competitive with transformers on language
  • Inference: O(1) per token — constant memory!
  • Best for: Long sequences, real-time, memory-constrained

Hybrid (Mamba + Attention)

  • Complexity: O(n) for most layers, O(n²) for attention layers
  • Quality: Best of both worlds
  • Used by: Jamba (AI21), Zamba (Zyphra)
  • Best for: Production systems needing quality + efficiency

💡 Key Insights

🎯 Selectivity is Key

Traditional SSMs use fixed matrices (A, B, C). Mamba makes them input-dependent — the model learns WHEN to remember and WHAT to forget. This is what makes it competitive with attention.

⚡ Hardware Efficiency

Mamba-2's SSD (Structured State Space Duality) shows that SSMs can be implemented as matrix multiplications, getting GPU-friendly parallelism while maintaining recurrence's efficiency.
