📚 What is GRPO?

GRPO (Group Relative Policy Optimization) is the training algorithm behind DeepSeek-R1, one of the most capable reasoning models. Instead of comparing "chosen vs rejected" responses (like DPO), GRPO generates N responses per prompt and ranks them against each other.

LGRPO = −𝔼[ min( rt · At, clip(rt, 1−ε, 1+ε) · At ) ] + β · KL(π ∥ πref)

Group Relative Policy Optimization

  • Input: Prompts only (no preference pairs needed)
  • Process: Generate N responses, rank them, optimize
  • Rewards: Automatic (e.g., math correctness)
  • Memory: ~1x model size (no value function)
  • Used by: DeepSeek-R1, DeepSeek-Math

Direct Preference Optimization

  • Input: Preference pairs (chosen vs rejected)
  • Process: Maximize margin between chosen and rejected
  • Rewards: Human preferences (expensive, noisy)
  • Memory: ~1x model size (no value function)
  • Used by: Zephyr, many open-source models

RL from Human Feedback (PPO)

  • Input: Human preferences → reward model
  • Process: PPO with learned reward model
  • Rewards: Learned reward model (can hack)
  • Memory: ~3x model size (policy + ref + reward + value)
  • Used by: ChatGPT, Claude, original InstructGPT

💡 Key Insights

🎯 The "Aha Moment"

During GRPO training, reasoning models develop behaviors they were never explicitly taught:

  • Self-correction ("Wait, let me re-check...")
  • Verification ("Let me verify this by...")
  • Multi-approach exploration
  • Breaking down complex problems

⚖️ Why Group Comparison?

Comparing N responses within a group provides:

  • Natural baseline (group mean)
  • No value function needed (saves 50% memory)
  • Robust to reward scale
  • Works with automatic rewards
