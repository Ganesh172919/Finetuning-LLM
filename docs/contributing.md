# SOTA AI — Contributing

## How to Contribute

### 1. Fork the Repository
```bash
git clone https://github.com/your-username/sota-ai.git
```

### 2. Create a Branch
```bash
git checkout -b feature/your-feature
```

### 3. Make Changes
- Follow the code style
- Add documentation
- Write tests

### 4. Submit Pull Request
- Describe your changes
- Reference any issues

---

## Code Style

### Documentation
- Every function has docstrings
- Explain concepts, not just code
- Include examples

### Comments
- Explain WHY, not WHAT
- Use educational style
- Include intuition

### Testing
- Test all functions
- Include edge cases

---

## What to Contribute

### High Priority
- New model implementations
- Better documentation
- More examples
- Bug fixes

### Medium Priority
- Performance optimizations
- Additional tests
- Tutorial content

### Low Priority
- Style changes
- Minor refactors

---

*All contributions welcome! See the issues for ideas.*
