QLoRA & Quantization

Fine-tune 65B models on a single 48GB GPU — 4-bit quantization + LoRA = accessible AI

What is QLoRA?

Zipping a Filing Cabinet

Imagine a massive filing cabinet (your model). Full fine-tuning needs you to copy the entire cabinet. QLoRA compresses it to 1/4 size (4-bit), then adds small sticky notes (LoRA adapters) for new knowledge. The cabinet stays compressed; only the notes are trained.

Why 4-bit?

A 65B model in FP16 needs 130GB. In 4-bit, it needs ~33GB — fits on a single A100! Combined with LoRA's small adapter, you can fine-tune it with just 48GB total GPU memory.

The Magic: NF4

Regular 4-bit quantization loses information unevenly. NF4 (NormalFloat4) is designed specifically for neural network weights, which follow a normal distribution. It places quantization levels where the data actually is.

How QLoRA Works

The 3 Innovations

1. NF4 Quantization: 4-bit data type optimized for normal distributions. Uses quantile-based mapping to place levels at optimal positions.

NF4 levels = quantiles of N(0,1) mapped to 16 bins

2. Double Quantization: Quantize the quantization constants themselves. FP32 → FP8 → NF4 saves an additional 0.37 bits per parameter.

3. Paged Optimizers: Use CPU memory for optimizer states when GPU memory runs out, via NVIDIA unified memory.

Total memory formula:

M = (P × b) / 8 + (P × b_r) / 8 + overhead

Where P = params, b = bits per param (4), b_r = LoRA bits (16), overhead ≈ 1-2GB

NF4 Quantization Deep Dive

Quantile Quantization

NF4 divides the normal distribution into 16 equal-probability bins. Each bin boundary is at a quantile of N(0,1). This ensures each quantization level represents equal probability mass, minimizing expected quantization error.

q_i = Φ⁻¹(i/16) for i = 0,1,...,15

Where Φ⁻¹ is the inverse CDF (probit function) of the standard normal.

Block-wise Quantization

Weights are grouped into blocks of 64. Each block gets its own scaling factor (absmax). This handles outliers: a single large weight doesn't waste precision for the rest.

w_q = round(w / block_scale × 7) → NF4 level

Double Quantization

The block scales (FP32) are themselves quantized to FP8 with block size 256. This saves 0.37 bits/param: (32/64 - 8/64) = 0.375. For 65B params, that's ~3GB saved.

Research Frontiers

GPTQ: Post-Training Quantization

Layer-wise quantization using Hessian information. Quantizes one column at a time, compensating errors in remaining columns. No fine-tuning needed — just calibration data.

AWQ: Activation-Aware Weight Quantization

Identifies salient weights by observing activations, not weights. Scales important channels before quantization to protect them. Better than GPTQ at 3-4 bits.

GGUF / llama.cpp

Runtime quantization format supporting Q2_K through Q8_0. Uses k-quant (mixed precision) with different quantization for attention vs FFN layers. Enables CPU inference.

QuIP# and AQLM

Research methods using incoherence processing and multi-codebook quantization. Achieve 2-bit quantization with minimal perplexity loss. Frontier of extreme compression.

Code Playground

QLoRA in Practice

from transformers import AutoModelForCausalLM, BitsAndBytesConfig
from peft import LoraConfig, get_peft_model, prepare_model_for_kbit_training

# 4-bit quantization config (QLoRA)
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",           # NormalFloat4
    bnb_4bit_compute_dtype=torch.float16,  # Compute in FP16
    bnb_4bit_use_double_quant=True,       # Double quantization
)

# Load model in 4-bit
model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-2-7b",
    quantization_config=bnb_config,
    device_map="auto",
)

# Prepare for QLoRA training
model = prepare_model_for_kbit_training(model)

# LoRA configuration
lora_config = LoraConfig(
    r=16,                    # Rank
    lora_alpha=32,            # Scaling factor
    target_modules=["q_proj", "v_proj", "k_proj", "o_proj"],
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM",
)

# Apply LoRA
model = get_peft_model(model, lora_config)
model.print_trainable_parameters()
# Output: trainable params: 4,194,304 || all params: 3,504,484,352 || 0.12%
