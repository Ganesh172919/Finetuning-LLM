LoRA Mathematics

Low-Rank Adaptation — fine-tune billion-parameter models with <1% of the parameters

What is LoRA?

The Shortcut Analogy

Imagine a highway (original model). Instead of rebuilding the entire highway, LoRA adds small side roads (low-rank matrices) that let traffic take new routes. The original highway stays frozen.

Why Not Full Fine-Tuning?

A 7B model has 7 billion parameters. Full fine-tuning needs gradients for all 7B params — that's 28GB+ just for optimizer states. LoRA trains only ~0.1% of parameters, using ~100x less memory.

The Core Idea

Instead of updating the full weight matrix W, LoRA decomposes the update into two small matrices: ΔW = A × B, where A is (d × r) and B is (r × d), with r ≪ d.

W' = W + ΔW = W + A · B

How LoRA Works

Low-Rank Decomposition

Given a pre-trained weight matrix W₀ ∈ ℝd×k, LoRA constrains the update:

W = W₀ + ΔW = W₀ + B · A

Where B ∈ ℝd×r and A ∈ ℝr×k, with rank r ≪ min(d, k).

Initialization: A ~ 𝒩(0, σ²), B = 0 (so ΔW = 0 at start)

Scaling: ΔW scaled by α/r to stabilize training

h = W₀x + (α/r) · B · A · x

Mathematical Properties

Rank and Expressiveness

Rank r controls the expressiveness of the update. A rank-r matrix can represent at most r linearly independent directions of change.

For a d×d matrix: full rank = d² params, rank-r = 2dr params.

Compression ratio = d² / (2dr) = d / (2r)

For d=4096, r=8: compression = 256× — from 16M to 65K params.

Spectral Analysis

The singular values of ΔW decay rapidly for tasks that modify only a few "directions" in weight space. LoRA exploits this — the top-r singular vectors capture most of the adaptation signal.

ΔW = U · Σ · Vᵀ ≈ U:,:r · Σ:r,:r · V:,:rᵀ

Merge and Inference

At inference time, LoRA adds zero latency by merging: W' = W₀ + (α/r)·B·A. This is a single matrix — no extra computation.

Research Frontiers

DoRA: Weight-Decomposed LoRA

Decomposes weight into magnitude and direction: W' = m · (W₀ + BA)/||W₀ + BA||. Adapts direction via LoRA, magnitude separately. Better on commonsense reasoning.

GaLore: Gradient Low-Rank Projection

Projects gradients into low-rank subspace instead of weights. Updates full weights but in a compressed gradient space. Memory savings without architecture changes.

LoRA+ : Learning Rate Disparity

Uses different learning rates for A and B: ηB = λ · ηA where λ > 1. Motivated by the asymmetry in how A and B gradients behave during training.

rsLoRA: Rank-Stabilized Scaling

Replaces α/r with α/√r scaling. This maintains gradient magnitude across different ranks, allowing higher ranks without instability.

Code Playground

Implement LoRA from Scratch

import numpy as np

class LoRALayer:
    """Low-Rank Adaptation layer."""

    def __init__(self, d_in, d_out, rank=4, alpha=1.0):
        # Frozen pre-trained weights
        self.W0 = np.random.randn(d_out, d_in) * 0.02

        # LoRA matrices
        self.A = np.random.randn(rank, d_in) * 0.01
        self.B = np.zeros((d_out, rank))  # Zero init!

        self.scaling = alpha / rank

    def forward(self, x):
        # h = W₀·x + (α/r)·B·A·x
        h_base = self.W0 @ x
        h_lora = self.scaling * (self.B @ (self.A @ x))
        return h_base + h_lora

    def merge(self):
        # Merge for zero-latency inference
        return self.W0 + self.scaling * (self.B @ self.A)

# Example: 4096→4096 layer, rank=8
layer = LoRALayer(4096, 4096, rank=8, alpha=16)
x = np.random.randn(4096)
out = layer.forward(x)
merged = layer.merge()

print(f"Original params: {layer.W0.size:,}")
print(f"LoRA params: {layer.A.size + layer.B.size:,}")
print(f"Compression: {layer.W0.size / (layer.A.size + layer.B.size):.1f}×)
