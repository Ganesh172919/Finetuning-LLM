Attention Mechanisms

The core innovation behind Transformers — understand how models "pay attention" to relevant information

What is Attention?

The Restaurant Analogy

Imagine you're at a noisy restaurant. Your brain "attends" to the person speaking to you, filtering out other conversations. Attention in AI works similarly — it learns which parts of the input to focus on.

Spotlight, Not Camera

A camera captures everything equally. A spotlight highlights what matters. Attention is like a spotlight — it gives more weight to important words and less to irrelevant ones.

Reading Comprehension

When answering "What color is the car?", your eyes jump to "car" in the text. Attention lets models do the same — dynamically focus on relevant parts of the input.

How Attention Works

Query, Key, Value

Attention uses three vectors for each token:

Query (Q): "What am I looking for?"

Key (K): "What do I contain?"

Value (V): "What information do I provide?"

Attention(Q, K, V) = softmax(Q · KT / √d) · V

The dot product Q·K measures similarity. Softmax converts to probabilities. The weighted sum of Values produces the output.

Multi-Head Attention & Complexity

Multi-Head Attention

Instead of one attention, run h parallel "heads" with different learned projections. Each head can attend to different types of relationships (syntactic, semantic, positional).

MultiHead(Q,K,V) = Concat(head₁,...,headₕ) · WO

Complexity Analysis

Time: O(n²·d) — quadratic in sequence length

Space: O(n²) for attention matrix

This quadratic cost is the main bottleneck for long sequences, motivating efficient attention variants like FlashAttention, Linear Attention, and sparse attention.

Attention Variants & Research Frontiers

Efficient Attention

FlashAttention: Tiled exact attention, O(N) memory

Linear Attention: O(N) via kernel trick

Sparse Attention: Local + global patterns

Multi-Query: Shared K,V across heads

Attention is Not Enough

Recent work shows attention heads have specialized roles: induction heads (in-context learning), retrieval heads (fact lookup), safety heads (refusal). Understanding head roles is key to interpretability.

Code Playground

self_attention.py
import numpy as np def self_attention(X, W_Q, W_K, W_V): """Compute self-attention for input sequence X.""" # Project input to Q, K, V Q = X @ W_Q # (seq_len, d_head) K = X @ W_K V = X @ W_V # Compute attention scores d_k = K.shape[-1] scores = Q @ K.T / np.sqrt(d_k) # (seq_len, seq_len) # Softmax (numerically stable) exp_scores = np.exp(scores - np.max(scores, axis=-1, keepdims=True)) weights = exp_scores / exp_scores.sum(axis=-1, keepdims=True) # Weighted sum of values output = weights @ V # (seq_len, d_head) return output, weights # Example: 6 tokens, 4-dimensional embeddings X = np.random.randn(6, 8) W_Q = np.random.randn(8, 4) W_K = np.random.randn(8, 4) W_V = np.random.randn(8, 4) output, attn_weights = self_attention(X, W_Q, W_K, W_V) print("Attention weights shape:", attn_weights.shape) print("Output shape:", output.shape)
> Attention weights shape: (6, 6)
> Output shape: (6, 4)

Test Your Understanding

1. What does the softmax in attention compute?

The exact values of each token
A probability distribution over how much to attend to each token
The gradient of the loss function
The position of each token in the sequence
Correct! Softmax converts raw attention scores into a probability distribution that sums to 1, telling the model how much to "attend" to each token.

2. Why do we divide by √d in the attention formula?

To make the computation faster
To reduce the number of parameters
To prevent softmax from saturating with large dot products
To make the output match the input dimension
Correct! Without scaling, large dot products push softmax into regions with tiny gradients, making training unstable. Dividing by √d keeps the variance of dot products at 1.

3. What is the main advantage of multi-head attention over single-head attention?

It uses less memory
Different heads can learn to attend to different types of relationships
It makes the model smaller
It removes the need for softmax
Correct! Each head has its own Q, K, V projections, allowing it to specialize in different patterns — some heads attend locally, some attend to specific syntactic roles, some capture long-range dependencies.
