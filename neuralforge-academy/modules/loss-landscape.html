🏔️ The Loss Landscape

Training a neural network is like navigating a mountain range in fog. The height represents the loss (error), and we want to find the lowest valley. The model's weights determine where we are on this terrain.

🎓 Conceptual Foundation

🧊 Like Finding the Lowest Spot in a Dark Room

Imagine you're blindfolded in a hilly field. You want to find the lowest spot. You can feel which way the ground slopes under your feet. So you always step downhill. That's what training does — it always tries to go where the error is lower!

Analogy: The "learning rate" is how big your steps are. Too big? You might step over the lowest spot and end up on another hill! Too small? It takes forever to get anywhere.

📐 Gradient Descent

The loss function measures how wrong the model is. It takes the model's weights as input and outputs a single number (the loss). Training means finding the weights that minimize this loss.

We compute the gradient (slope) of the loss with respect to each weight. Then we update weights in the opposite direction of the gradient — moving "downhill."

⚙️ Optimization Dynamics

The loss landscape is non-convex with many local minima, saddle points, and plateaus. Modern optimizers use momentum and adaptive learning rates to navigate this terrain.

  • SGD: Noisy, can escape shallow minima but slow to converge
  • Momentum: Accumulates velocity, faster convergence
  • Adam: Adaptive per-parameter learning rates
  • AdamW: Adam with decoupled weight decay (standard for LLMs)

⚠️ Warning: Learning Rate Too High
When the learning rate is too high, the optimizer overshoots the minimum. The loss oscillates wildly or even increases. This is called "divergence."

📐 Mathematical Foundation

Gradient Descent Update Rule

θ_{t+1} = θ_t - η · ∇L(θ_t)

Where θ = model weights, η = learning rate, ∇L = gradient of loss

Momentum Update

v_t = β · v_{t-1} + ∇L(θ_t)
θ_{t+1} = θ_t - η · v_t

Momentum accumulates past gradients, smoothing out noise and accelerating convergence.

Adam Update

m_t = β₁ · m_{t-1} + (1 - β₁) · g_t
v_t = β₂ · v_{t-1} + (1 - β₂) · g_t²
θ_{t+1} = θ_t - η · m̂_t / (√v̂_t + ε)

Adam adapts learning rates per-parameter using first (m) and second (v) moment estimates.

💻 Code Implementation

import numpy as np class SGD: """Stochastic Gradient Descent with optional momentum.""" def __init__(self, lr=0.01, momentum=0.0): self.lr = lr self.momentum = momentum self.velocity = None def step(self, params, grads): if self.velocity is None: self.velocity = [np.zeros_like(p) for p in params] for i in range(len(params)): # Update velocity with momentum self.velocity[i] = self.momentum * self.velocity[i] + grads[i] # Update parameters params[i] -= self.lr * self.velocity[i] class Adam: """Adam optimizer with bias correction.""" def __init__(self, lr=0.001, beta1=0.9, beta2=0.999, eps=1e-8): self.lr = lr self.beta1, self.beta2, self.eps = beta1, beta2, eps self.m, self.v, self.t = None, None, 0 def step(self, params, grads): if self.m is None: self.m = [np.zeros_like(p) for p in params] self.v = [np.zeros_like(p) for p in params] self.t += 1 for i in range(len(params)): # Update biased first and second moment self.m[i] = self.beta1 * self.m[i] + (1 - self.beta1) * grads[i] self.v[i] = self.beta2 * self.v[i] + (1 - self.beta2) * grads[i]**2 # Bias correction m_hat = self.m[i] / (1 - self.beta1**self.t) v_hat = self.v[i] / (1 - self.beta2**self.t) # Update parameters params[i] -= self.lr * m_hat / (np.sqrt(v_hat) + self.eps)
